# 低波动操作跨周期过滤策略报告

> **适用标的**: 3690.HK（美团）及类似波动特征的港股
> **策略框架**: 4周期（15分钟/60分钟/日线/周线）Schmitt Trigger + 预测曲线系统
> **研究日期**: 2026-06-20
> **现状分析来源**: `streamlit_app.py` (1680行)、`config/3690_HK.json`、`docs/STPP_策略文档.md`
>
> **缩写约定**: ST = Schmitt Trigger, SP = Savitzky-Golay滤波价格, v = dSP/dt (速度), a = dv/dt (加速度)

---

## 摘要

### 问题定义

当前3690.HK的4周期交易系统在单周期内通过Schmitt Trigger自适应死区实现信号过滤，但存在一个结构性缺陷：**每个周期的信号独立产生、独立评估，不利用跨周期信息**。具体场景如下：

> 以60分钟线做多为例：
> 1. 日线级别是否有大周期趋势的反转确认？
> 2. 60分钟线做空时间是不是特别短（暗示空头无力，多头可信）？

这两个问题在单周期框架内无法回答。本报告提出**三层跨周期过滤架构**，在任何单周期Schmitt信号产生后，逐层检查长周期趋势确认、信号持续时间质量、以及多维度一致性，实现从"4视图独立展示面板"到"多周期协同决策系统"的升级。

### 三层过滤架构

```
单周期Schmitt信号 Sig=+1 或 Sig=−1
       │
       ▼
┌─────────────────────────────────────────┐
│ 第一层: 长周期趋势确认 (Trend Confirmation)  │
│  日线5态分类 → 反转检测 → 新鲜度衰减         │
│  周线宏观约束（CAP仓位上限）                 │
│  输出: trend_score ∈ [0, 1]               │
└──────────────────┬──────────────────────┘
                   │ 通过 (≥0.3)
                   ▼
┌─────────────────────────────────────────┐
│ 第二层: 信号持续时间质量 (Duration Quality)   │
│  前序反向信号持续时间评估                     │
│  动态阈值 → quality_boost / penalty         │
│  一票否决: 长周期方向与本周期相反 → 直接过滤  │
│  输出: quality_factor ∈ [0.7, 1.5]         │
└──────────────────┬──────────────────────┘
                   │ 通过（未被一票否决）
                   ▼
┌─────────────────────────────────────────┐
│ 第三层: 多维一致性检查 (Consistency Check)    │
│  6维度评分卡 (波动率/价格/共振/背离/频率/预测)  │
│  输出: consistency_score ∈ [0, 1]         │
└──────────────────┬──────────────────────┘
                   │
                   ▼
        综合评分 → 决策阈值 → 仓位映射
```

### 核心结论

1. **跨周期信息是假信号过滤的最有效武器**。单周期内Schmitt Trigger的死区机制只能过滤噪声，无法识别"逆大趋势的小级别信号"，而后者是低波动操作中主要的亏损来源。第一层趋势确认 + 第二层一票否决，预计可过滤掉40-55%的假信号。

2. **前序信号持续时间包含预测信息**。当60分钟线前序空头信号持续时间异常短（低于动态阈值），说明空头无力，后续多头信号的可靠性显著提升。反之，前序信号持续时间异常长，可能意味着趋势深厚，反转是诱多。

3. **多维度一致性评分提供仓位精细化管理**。不仅输出"做/不做"，更输出0-1的质量评分，映射到0/25%/50%/100%四级仓位。

## 1. 问题定义

### 1.1 什么是"低波动操作"

"低波动操作"指在标的处于低波动率环境时执行的交易策略。3690.HK作为港股大盘科技股，其日内波动特性呈现明显的波动率聚集现象——70%的交易日波动率处于中低区间，但30%的交易日出现波动率突变。

低波动环境下的两个关键挑战：

- **信号稀疏化**：Schmitt Trigger的自适应死区 `ε_t = k_ε · max(σ_t(v), σ_min)` 在低波动时自然收窄，但低波动也意味着信号强度整体下降，真实信号与噪声的区分度降低。
- **假突破高发**：低波动阶段末端常伴随方向性突破，但单周期框架无法区分"真正的趋势启动"和"低波动中的随机脉冲"。

当前系统的4周期配置（15分钟/60分钟/日线/周线）参数完全统一：`ke=0.15, sm=0.05, ew=60`。`N_EWMA=60` 在15分钟线上半衰期约21 bars（5.25小时），在周线上半衰期约21 bars（5个月），物理含义截然不同（参见《3690.HK 多周期过滤算法深度研究报告》T2发现）。

### 1.2 为什么单周期过滤不够

单周期过滤的局限性体现在三个层面：

| 层面 | 单周期能做到 | 单周期做不到 |
|------|-------------|-------------|
| 噪声过滤 | 通过SG平滑 + Schmitt死区过滤随机波动 | 区分"与趋势同向的噪声"和"真正的趋势信号" |
| 信号产生 | 基于该周期的v/a独立判断 | 参考更大周期（或更小周期）的趋势状态 |
| 仓位管理 | 统一2%止损 | 根据跨周期信息调整仓位大小 |

以60分钟线做多为例，单周期只能判断：在60分钟尺度上，a > ε_t 且 v > 0。它无法回答：
- 日线是否处于上升趋势中？如果日线处于下降趋势，60分钟的做多信号大概率是反弹而非反转。
- 周线是否支持做多？如果周线处于下降趋势，60分钟的多头信号面临更强的宏观阻力。

### 1.3 跨周期信息的价值

跨周期信息包含三种类型：

1. **趋势方向信息**（第一层）：更大周期的趋势方向作为小周期信号的"背景色"。当60分钟出现做多信号时，日线的趋势方向决定了信号的"顺势/逆势"属性。

2. **信号结构信息**（第二层）：信号的持续时间、持续性模式包含关于信号强度的信息。例如，一段异常短的空头信号后跟随的多头信号，往往比一段正常持续时间的空头信号后的多头信号更可靠。

3. **一致性信息**（第三层）：多个周期在波动率环境、价格位置、背离状态等方面是否一致。一致性越高，信号越可靠。

**核心洞察**：跨周期信息不是"锦上添花"，而是低波动操作中区分盈亏的**关键差异因素**。在低波动环境下，单周期信号的SNR本就偏低，跨周期的一次性过滤可以大幅提升总体决策质量。

### 2.2 三层之间的关系

三层过滤是**顺序执行、逐层收紧**的结构：

| 层 | 角色 | 失败后果 | 通过后继续 |
|----|------|---------|-----------|
| 第一层: 趋势确认 | **门卫（Gatekeeper）** | 直接过滤，不进入后续层 | 进入第二层 |
| 第二层: 持续时间 | **质量调节器（Modulator）** | 一票否决时直接过滤；否则输出调节因子 | 携带调节因子进入第三层 |
| 第三层: 一致性 | **精细评分（Scorer）** | 输出0-1评分 | 汇总为最终决策分数 |

**关键设计原则**：每层只做一个判断。第一层判断"大环境对不对"，第二层判断"这个信号的对手强不强"，第三层判断"所有周期是否一致"。避免单层做多个判断导致逻辑耦合和调参困难。

### 2.3 过滤优先级

信号过滤的优先级从上到下递减：

1. **一票否决**（最高优先级）：长周期方向与本周期信号方向相反 → 直接过滤。这是第二层的规则，但在执行顺序上优先于一切其他规则。
2. **趋势降级**（高优先级）：周线方向与信号方向相反 → 无论其他评分多高，仓位不超过1/4。
3. **综合评分过滤**（中优先级）：三层综合评分 < 0.3 → 过滤。
4. **仓位调整**（低优先级）：综合评分在0.3-1.0之间 → 按区间映射仓位。

## 3. 第一层：长周期趋势确认

### 3.1 日线趋势状态分类

日线5态分类器基于日线Schmitt Trigger信号和滤波价格的速度(v)、加速度(a)：

```
日线状态分类（每bar更新）:
┌──────────────┬────────────────────────────────────────────┐
│ 状态           │ 判定条件                                    │
├──────────────┼────────────────────────────────────────────┤
│ STRONG_UP    │ Sig==+1 且 a > +ε (加速上涨)                 │
│ WEAK_UP      │ Sig==+1 且 a ∈ [-ε, +ε] (上涨减速或横盘)     │
│ NEUTRAL      │ Sig==0 或 价格位于布林带中轨±0.5σ内          │
│ WEAK_DOWN    │ Sig==−1 且 a ∈ [-ε, +ε] (下跌减速或横盘)     │
│ STRONG_DOWN  │ Sig==−1 且 a < −ε (加速下跌)                 │
└──────────────┴────────────────────────────────────────────┘
```

**为什么用5态而非3态**：
- 3态（UP/NEUTRAL/DOWN）丢失了趋势强度信息。WEAK_UP和STRONG_UP对小周期信号的支持力度不同。
- WEAK_UP状态下60分钟的做多信号需更高确认门槛（因为日线上涨动能正在衰减）。
- 5态为后续的"趋势反转检测"提供了更精细的状态空间。

伪代码：

```
def classify_daily_trend(daily_sig, daily_a, daily_eps, daily_price, bb_mid, bb_std):
    if daily_sig == +1 and daily_a > daily_eps:
        return "STRONG_UP"
    if daily_sig == +1 and abs(daily_a) <= daily_eps:
        return "WEAK_UP"
    if daily_sig == -1 and daily_a < -daily_eps:
        return "STRONG_DOWN"
    if daily_sig == -1 and abs(daily_a) <= daily_eps:
        return "WEAK_DOWN"
    return "NEUTRAL"
```

### 3.2 趋势反转检测

使用"A+C组合"检测日线级别的趋势反转：

- **A信号（加速度反转 Acceleration Reversal）**：`a_日线` 上穿0，且 `v_日线` 方向与新方向一致。这是**早期信号**，捕获趋势反转的初始动量变化。
- **C信号（Schmitt确认 Confirmation）**：`Sig_日线` 从0翻转为±1。这是**确认信号**，Schmitt Trigger的双重条件（加速度+速度）已确认。

```
反转检测逻辑（以空转多为例）:
  A信号触发: a_日线[i] > 0 且 v_日线[i] > 0 （加速度转正 + 速度为正）
  C信号触发: Sig_日线[i] == +1 且 Sig_日线[i-1] != +1 （Schmitt确认翻转）

  反转确认条件:
    - A+C同时触发 → 强反转确认
    - 仅A触发 → 弱反转，仅记录不确认
    - 仅C触发 → 标准反转，正常确认
```

**为什么是A+C组合**：
- 单独使用A信号：灵敏度高但噪声多（加速度可能在零附近反复交叉）。
- 单独使用C信号：确认可靠但滞后大（Schmitt的双重条件需要累积足够证据）。
- A+C组合：A提供及时性，C提供可靠性，两者互补。

### 3.3 反转新鲜度衰减

反转信号的价值随时间衰减。今天出现的日线反转，对今天60分钟做多的支持力度强；3天前的反转，支持力度显著下降。

**指数衰减函数**：

```
freshness = exp(-λ × bars_ago / half_life)

参数:
  λ = 1.5         衰减速率系数
  half_life = 3   半衰期（bars）
  bars_ago        距离最近一次反转确认的日线bar数
```

衰减曲线：

```
bars_ago:      0      1      2      3      4      5      6
freshness:   1.00   0.61   0.37   0.22   0.14   0.08   0.05
```

**为什么用指数衰减**：
- 反转的预测价值不是线性的。第1天和第2天的信息价值差距远大于第5天和第6天。
- 指数衰减更符合市场的"信息消化"特征——反转发生后，市场在最初几个bar内快速消化新信息。

### 3.4 周线宏观约束

周线作为最长周期，提供宏观约束。周线本身不产生"是否做"的判断，而是**施加仓位上限（CAP）**：

```
周线状态分类:
  MACRO_UP:   Sig_周线 == +1
              → CAP: 全仓（允许100%仓位）
  NEUTRAL:    Sig_周线 == 0 或 数据不足
              → CAP: 半仓（最多50%仓位），需日线额外确认
  MACRO_DOWN: Sig_周线 == −1
              → CAP: 1/4仓（最多25%仓位），需强确认（综合确认权重 > 0.7）
```

**为什么周线只做约束不做决策**：
- 周线bar数量少（`n=90`），统计可靠性不如日线和60分钟线。
- 周线信号变化慢，不适合做频繁的入场/离场判断。
- 周线的最佳用途是设定"大方向"——顺周线方向加大仓位，逆周线方向缩小仓位。

### 3.5 综合确认权重

第一层输出一个 `trend_score ∈ [0, 1]`：

```
trend_score = 0.40 × daily_state_score
            + 0.25 × freshness
            + 0.25 × weekly_state_score
            + 0.10 × price_position_score

各项计算:
  daily_state_score:
    STRONG_UP ∧ 信号做多   → 1.0
    WEAK_UP ∧ 信号做多     → 0.7
    NEUTRAL                → 0.5
    WEAK_DOWN ∧ 信号做多   → 0.2
    STRONG_DOWN ∧ 信号做多 → 0.0
    (做空信号同理，取反向映射)

  freshness = exp(-1.5 × bars_ago / 3)

  weekly_state_score:
    MACRO_UP ∧ 信号做多    → 1.0
    NEUTRAL                → 0.5
    MACRO_DOWN ∧ 信号做多  → 0.2

  price_position_score:
    价格相对布林带位置在有利区间 → 1.0
      (做多: 价格在下半区; 做空: 价格在上半区)
    价格在中性区间              → 0.5
    价格在不利区间              → 0.2
```

**阈值决策**：

| trend_score | 决策 | 说明 |
|-------------|------|------|
| < 0.3 | **FILTER** | 大环境不配合，直接过滤信号 |
| 0.3 - 0.6 | **HALF_CAP** | 允许半仓，进入第二层 |
| > 0.6 | **FULL_CAP** | 允许全仓（受周线CAP约束），进入第二层 |

**权重设计的依据**：
- `daily_state_score` 占40%：日线是60分钟信号最直接的"上级周期"，权重最高。
- `freshness` 占25%：反转的时效性至关重要——3天前的反转比今天的反转支持力度低4倍多。
- `weekly_state_score` 占25%：周线提供宏观约束，但bar数量少，不宜占太高。
- `price_position_score` 占10%：价格位置是辅助信息，独立预测能力有限。

## 4. 第二层：信号持续时间质量

### 4.1 前序信号质量传递规则

核心思想：**前一个反向信号的持续时间包含了关于当前信号质量的信息**。

以60分钟线做多为例（前序信号为空头）：

```
规则1（空头无力，多头可信）:
  前序空头持续时间 < threshold_short → quality_boost = 1.0 ~ 1.5
  逻辑: 空头连基本持续时间都达不到就被逆转，说明空方力量薄弱，
        新生的多头信号可信度提升。

规则2（正常范围，不调整）:
  threshold_short ≤ 前序空头持续时间 ≤ threshold_long → quality_factor = 1.0
  逻辑: 前序空头持续时间在正常范围内，无法从中提取额外信息。

规则3（趋势深厚，反转可能是诱多）:
  前序空头持续时间 > threshold_long → quality_penalty = 0.7
  逻辑: 空头信号持续了远超正常范围的时间，说明空头趋势深厚，
        此时的反转大概率是短期反弹（诱多），而非真正的趋势翻转。

规则4（无前序信号）:
  当前信号是窗口内的第一个信号 → quality_factor = 0.8
  逻辑: 缺乏前序参考信息，采取保守默认值。
```

**为什么规则1和规则3不对称**：
- 信息不对称：空头信号"异常短"和"异常长"提供的信息量不同。异常短更可靠（空头确实无力），异常长的惩罚更重（趋势深厚时反转风险大）。
- 市场非对称性：港股市场下跌通常比上涨更快（恐慌性抛售），因此空头信号的"正常持续时间"本身就比多头信号短。规则必须考虑这种非对称性。

### 4.2 动态阈值自适应

阈值不是固定值，而是基于该周期信号持续时间的近期分布动态计算：

```
动态阈值计算:
  μ = 过去N个该周期同向信号持续时间的均值  (建议 N=20)
  σ = 标准差

  threshold_short = max(3, μ - 1.5σ)
  threshold_long  = μ + 2σ

地板保护:
  threshold_short 最小值 = 3 (防止阈值过低导致信号过密)
```

**为什么是 μ - 1.5σ 和 μ + 2σ**：
- `μ - 1.5σ`（短阈值）：约覆盖正态分布下6.7%的左侧尾部。选择1.5σ而非2σ，是因为我们更关注"异常短"的检测——希望捕捉更多空头无力的信号，不过于保守。
- `μ + 2σ`（长阈值）：约覆盖正态分布下2.3%的右侧尾部。选择2σ而非1.5σ，是因为"异常长"的惩罚较重（直接降到0.7），我们希望这个判定足够保守，只在确实异常时才触发。

**自适应的重要性**：
- 不同周期（60分钟 vs 15分钟）信号持续时间天然不同。
- 同一周期在不同市场阶段（高波动 vs 低波动）信号持续时间也会变化。
- 动态阈值确保规则始终以"近期正常"为基准，适应市场变化。

### 4.3 双向持续时间比值

除了前序信号的绝对持续时间，当前信号与前序信号的持续时间**比值**也包含信息：

```
duration_ratio = dur_current / dur_prev_opposite

解读:
  ratio < 0.3  → 当前信号尚短，前序可能未充分消耗 → 强信号
  0.3 ≤ ratio ≤ 1.0 → 正常范围内 → 不调整
  ratio > 2.0 → 当前信号已远超前序 → 可能接近结束 → quality × 0.85
```

**使用场景**：该比值主要用于辅助判断"当前信号是否已经延长到不合理程度"，而非独立决策。它和动态阈值的规则4配合使用——当ratio > 2.0且前序在正常范围时，给一个温和的质量折扣。

### 4.4 与长周期确认的协同（一票否决）

**一票否决规则（最高优先级）**：

```
当同时满足以下条件时，直接FILTER信号:
  条件1: 长周期(日线或周线)的Sig方向与本周期信号方向相反
  条件2: 长周期Sig持续时间 ≥ 3 bars (确保不是噪声)

具体场景:
  60分钟Sig=+1 且 日线Sig=−1 且 日线Sig_dur ≥ 3 → **一票否决**
  60分钟Sig=−1 且 日线Sig=+1 且 日线Sig_dur ≥ 3 → **一票否决**
```

**为什么一票否决优先于所有质量提升**：
- 与大趋势对抗是小周期交易中最危险的错误。无论小周期信号看起来多好，逆大趋势的胜率都显著降低。
- 一票否决的触发条件是"长周期持续时间≥3 bars"——确保不是因为噪声导致的瞬时反向，而是确实的持续趋势。

**一票否决 vs 第一层趋势确认的区别**：
- 第一层趋势确认使用 `trend_score` 的连续评分，允许"半仓"等柔性处理。
- 一票否决是二值化的硬过滤——只要长周期方向明确相反，直接拒绝，不留余地。
- 两者的触发条件不同：第一层可能因为趋势不明确给低分（但不一定否决），一票否决只在方向明确相反时才触发。

## 5. 第三层：多维一致性检查

### 5.1 波动率环境（权重 10%）

评估当前市场波动率是否处于"正常"状态，以及是否存在波动率突变。

```
sigma_v_score = 0.5 × sigma_v_percentile_score + 0.5 × regime_stability_score

sigma_v_percentile_score (基于252天EWMA波动率分位数):
  percentile ∈ [20, 80]  → 1.0  (正常波动，信号可靠)
  percentile ∈ [10, 20) ∪ (80, 90] → 0.7
  percentile ∈ [0, 10) ∪ (90, 100] → 0.3  (极端波动，过滤)

regime_stability_score (波动率突变检测):
  |sigma_v[i] - sigma_v[i-1]| / sigma_v[i-1] < 0.2 → 1.0 (稳定)
  < 0.5 → 0.7
  ≥ 0.5 → 0.3 (突变，谨慎)
```

**为什么波动率突变要降分**：
- 波动率突变（regime shift）意味着市场正在进入新的波动状态。
- Schmitt Trigger的EWMA波动率估计需要~21 bars才能适应新状态。
- 在过渡期内，当前σ估计可能失真，信号的可靠性降低。

### 5.2 价格位置（权重 15%）

评估当前价格在最近走势中的相对位置，三个子维度：

```
price_position_score = 0.4 × bb_position + 0.3 × range_percentile + 0.3 × ma_deviation

bb_position (布林带位置，基于20-bar布林带):
  做多: 价格在下轨附近（0-25%分位）→ 1.0 (低位买入有利)
         价格在中轨附近（25-75%）  → 0.7
         价格在上轨附近（75-100%） → 0.3 (高位买入风险)
  做空: 反向映射

range_percentile (近期高低点分位，基于最近20-bar):
  做多: 分位 < 30% → 1.0 (相对低位)
         30-70%     → 0.6
         > 70%      → 0.3 (相对高位)
  做空: 反向映射

ma_deviation (均线偏离):
  基于60-bar EMA偏差：
  |deviation| / sigma_price < 1.0 → 1.0 (正常范围内)
  < 2.0 → 0.7
  ≥ 2.0 → 0.3 (远离均线，均值回归压力大)
```

### 5.3 多周期共振（权重 25%）

这是第三层中权重最高的维度，直接评估4周期的信号方向是否一致。

```
resonance_score = Σ (weight[i] × direction_match[i])  for i in [15min, 60min, daily, weekly]

周期权重（按周期长度递增）:
  15分钟:  0.15   (最短周期，噪声多，权重最低)
  60分钟:  0.25   (交易周期本身，中等权重)
  日线:    0.30   (上级周期，权重高)
  周线:    0.30   (宏观周期，权重最高)

direction_match[i]:
  该周期Sig与本交易信号方向一致 → 1.0
  该周期Sig=0                     → 0.5
  该周期Sig与本交易信号方向相反    → 0.0
```

**为什么周线权重和日线相同（0.30）而非更高**：
- 周线bar少，统计可靠性不如日线。
- 但周线方向代表宏观趋势，一旦确认，其影响是决定性的（体现为第二层的一票否决）。
- 因此共振评分中给日线和周线相同权重，但周线在一票否决中发挥独特作用。

### 5.4 背离检测（权重 20%）

检测价格走势与技术指标的背离，是经典技术分析中最可靠的信号之一。

```
divergence_score = 0.5 × top_divergence + 0.5 × cross_divergence

top_divergence (顶/底背离):
  顶背离 (bearish_divergence):
    价格创新高 + 当前Sig_状态未跟随创新高 (或a减速)
    → 做多信号时降分 (divergence_score += 0.2)
  底背离 (bullish_divergence):
    价格创新低 + 当前Sig_状态未跟随创新低 (或a减速)
    → 做空信号时降分 (divergence_score += 0.2)
  无背离 → 1.0

cross_divergence (跨周期背离):
  小周期趋势与长周期价格位置背离:
    60分钟Sig=+1 但 日线价格接近布林上轨 → 部分背离 → 0.6
    60分钟Sig=+1 但 日线价格趋势向下（SP下移） → 完全背离 → 0.3
  无背离 → 1.0
```

### 5.5 信号频率（权重 15%）

高频翻转的信号通常不可靠——好的趋势信号应该有一定的持续性。

```
frequency_score = 0.5 × flip_rate_score + 0.5 × max_duration_score

flip_rate_score (翻转率):
  过去20 bars内Sig翻转次数:
    0-2次  → 1.0  (低频翻转，趋势稳定)
    3-4次  → 0.7
    5-6次  → 0.4
    > 6次  → 0.2  (高频翻转，震荡市，信号不可靠)

max_duration_score (最大持续期):
  该周期历史上最大的Sig持续时长，作为当前信号持续能力的参考:
    max_dur ≥ 10 bars → 1.0 (该周期历史上能维持长趋势)
    5-9 bars           → 0.7
    < 5 bars           → 0.4 (该周期从未能维持趋势，信号存疑)
```

**为什么信号频率维度存在**：
- 如果该周期过去20 bars内翻转超过6次，说明当前处于震荡市。震荡市中Schmitt信号的信噪比极低。
- 该维度与第二层"前序信号持续时间"不同——第二层关注前序信号的个体特征，第五维度关注信号模式的整体统计特征。

### 5.6 预测质量（权重 15%）

评估STPP系统中拟合曲线的前向预测质量。

```
prediction_score = 0.4 × r_squared + 0.3 × curvature_quality + 0.3 × direction_match

r_squared:
  抛物线拟合的R²:
    R² > 0.8  → 1.0
    R² ∈ [0.5, 0.8] → 0.7
    R² < 0.5 → 0.4

curvature_quality (曲率合理性):
  方向匹配: 抛物线的开口方向与信号方向一致 → 1.0
            (做多时a>0(向上开口)为优，做空时a<0(向下开口)为优)
  不一致 → 0.4

direction_match (预测方向一致性):
  预测段(N=8 bars)的方向与当前信号方向一致 → 1.0
  预测段方向模糊(接近水平) → 0.6
  预测段方向与信号方向相反 → 0.2
```

## 6. 综合评分与决策

### 6.1 综合评分公式

```
final_score = trend_score × quality_factor × consistency_score

其中:
  trend_score       ∈ [0, 1]   (第一层输出)
  quality_factor    ∈ [0.7, 1.5] (第二层输出)
  consistency_score ∈ [0, 1]   (第三层输出)

consistency_score = 0.10 × sigma_v_score
                  + 0.15 × price_position_score
                  + 0.25 × resonance_score
                  + 0.20 × divergence_score
                  + 0.15 × frequency_score
                  + 0.15 × prediction_score
```

**为什么用乘法而非加权平均**：
- 三层过滤是串联结构——任何一层的问题都应该拉低总分。
- 加权平均允许"一层极差但其他层好"获得中等分数，这违反了过滤的初衷。
- 乘法确保：一票否决（quality_factor被设为0）直接清零；trend_score很低时，即使其他层好也无法获得高分。

### 6.2 决策阈值与仓位映射

```
决策矩阵:
  final_score ≤ 0.3         → REJECT (不交易)
  0.3 < final_score ≤ 0.5   → CAUTION (25%仓位，需人工确认)
  0.5 < final_score ≤ 0.7   → HALF (50%仓位)
  final_score > 0.7          → FULL (100%仓位，受周线CAP约束)

周线CAP覆盖规则:
  如果周线状态 == MACRO_DOWN:
    无论final_score如何，最大仓位 = 25%
  如果周线状态 == NEUTRAL:
    无论final_score如何，最大仓位 = 50%
```

仓位与分数映射的设计逻辑：

| final_score 范围 | 仓位 | 场景描述 |
|-----------------|------|---------|
| > 0.7 | 100% (受CAP) | 三层全部绿灯：趋势支持、信号质量高、多周期共振 |
| 0.5 - 0.7 | 50% | 大部分维度通过，但存在一些不确定性 |
| 0.3 - 0.5 | 25% + 人工 | 边缘情况：部分维度支持但整体信心不足 |
| ≤ 0.3 | 0% | 任一关键维度不通过，不交易 |

### 6.3 特殊规则（降级、一票否决）

除了综合评分外，以下特殊规则独立触发，覆盖综合评分：

```
规则汇总:
┌──────────────────────────────────────────────────────────────┐
│ 规则                          │ 触发条件            │ 动作     │
├──────────────────────────────────────────────────────────────┤
│ 一票否决(长周期反向)           │ 日线/周线Sig方向相反 │ → REJECT │
│                               │ 且持续时间≥3 bars   │          │
├──────────────────────────────────────────────────────────────┤
│ 周线降级                      │ 周线 MACRO_DOWN     │ CAP=25%  │
├──────────────────────────────────────────────────────────────┤
│ 新鲜度过期                     │ freshness < 0.15    │ 强制降级  │
│                               │ (bars_ago > 5)      │ 一档     │
├──────────────────────────────────────────────────────────────┤
│ 波动率突变                     │ regime_stability    │ 强制降级  │
│                               │ _score < 0.5        │ 一档     │
└──────────────────────────────────────────────────────────────┘
```

**"强制降级一档"的含义**：FULL → HALF, HALF → CAUTION, CAUTION → REJECT。

### 7.2 场景演练

#### 场景1：信号通过 — 全仓做多

**市场背景**：3690.HK经过两周调整后，日线出现A+C组合反转，周线维持多头。

```
各层输入:
  Layer 1:
    daily_state   = WEAK_UP (日线Sig=+1, a略低于ε)
    最近反转: bars_ago = 1, freshness = exp(-1.5×1/3) = 0.61
    周线: MACRO_UP
    价格位置: 布林下轨附近(20%分位) → 有利
    
    trend_score = 0.40×0.7 + 0.25×0.61 + 0.25×1.0 + 0.10×1.0
                = 0.28 + 0.15 + 0.25 + 0.10 = 0.78 ✅ (>0.6, FULL_CAP)

  Layer 2:
    前序空头持续时间: 4 bars
    动态阈值: μ=9, σ=3 → threshold_short = max(3, 9-4.5) = 4.5
    dur_prev(4) < threshold_short(4.5) → 规则1触发!
    quality_boost = 1.3 (空头无力，多头可信)
    双向比值: dur_current(刚开始)/4 < 0.3 → ratio检查留待后续
    
    quality_factor = 1.3

  Layer 3:
    sigma_v: 分位45%, 稳定 → 1.0 × 0.10 = 0.10
    price: BB 20% + 分位25% + dev 0.8σ → (1.0×0.4+1.0×0.3+1.0×0.3)=1.0 × 0.15 = 0.15
    resonance: 15min(+1), 60min(+1), 日线(+1), 周线(+1) → (0.15×1.0+0.25×1.0+0.30×1.0+0.30×1.0)=1.0 × 0.25 = 0.25
    divergence: 无背离 → 1.0 × 0.20 = 0.20
    frequency: 翻转2次, max_dur=12 → (1.0×0.5+1.0×0.5)=1.0 × 0.15 = 0.15
    prediction: R²=0.82, 曲率向上, 方向一致 → (1.0×0.4+1.0×0.3+1.0×0.3)=1.0 × 0.15 = 0.15
    
    consistency_score = 0.10+0.15+0.25+0.20+0.15+0.15 = 1.00

最终决策:
  final_score = 0.78 × 1.3 × 1.00 = 1.014 → 1.0 (cap)
  决策: **FULL** (受周线MACRO_UP支持, 全仓)
  
  解释: 日线刚刚反转(WEAK_UP=上涨初期) + 新鲜度高(昨天反转) + 
        周线多头 + 前序空头无力 + 4周期全部共振 →
        这是一个高确定性的做多机会。
```

#### 场景2：信号被过滤 — 一票否决

**市场背景**：3690.HK日线处于明确下降趋势中，60分钟出现做多信号。

```
各层输入:
  CHECK 0 (一票否决):
    日线Sig=−1, 持续时间=7 bars (≥3) ✓
    日线方向(−1) 与 60分钟信号(+1) 相反 ✓
    → **一票否决触发**
    
  **REJECT** — 不进入后续层

  解释: 日线已持续做空7天，这是一个明确的下降趋势。
        60分钟的做多信号在日线空头趋势中大概率只是下跌中继反弹。
        无论60分钟信号看起来多好，必须过滤。
        
  如果交易者坚持要做: 需要日线先出现A信号（加速度反转）
  或C信号（Schmitt翻转为+1），确认日线反转后再考虑。
```

#### 场景3：边缘信号 — 降级到CAUTION

**市场背景**：日线处于中性状态，反转发生已5天，前序空头持续正常。

```
各层输入:
  Layer 1:
    daily_state   = NEUTRAL (日线Sig=0)
    最近反转: bars_ago = 5, freshness = exp(-1.5×5/3) = 0.08
    周线: NEUTRAL
    价格位置: 布林中轨附近 → 中性
    
    trend_score = 0.40×0.5 + 0.25×0.08 + 0.25×0.5 + 0.10×0.5
                = 0.20 + 0.02 + 0.125 + 0.05 = 0.395
    → HALF_CAP (但freshness<0.15触发降级) → 实际CAP降为CAUTION

  Layer 2:
    前序空头持续时间: 8 bars
    动态阈值: μ=9, σ=3 → threshold_short=4.5, threshold_long=15
    在正常范围 → quality = 1.0
    quality_factor = 1.0

  Layer 3:
    sigma_v: 分位70%, 稳定 → 1.0 × 0.10 = 0.10
    price: BB中轨, 分位50%, dev 0.5σ → (0.7+0.6+1.0)/3 ≈ 0.77 × 0.15 = 0.12
    resonance: 15min(0), 60min(+1), 日线(0), 周线(0) → (0.15×0.5+0.25×1.0+0.30×0.5+0.30×0.5)=0.625 × 0.25 = 0.16
    divergence: 无背离 → 1.0 × 0.20 = 0.20
    frequency: 翻转4次, max_dur=7 → (0.7+0.7)/2=0.7 × 0.15 = 0.11
    prediction: R²=0.6, 曲率略向下(逆做多方向) → (0.7+0.4+1.0)/3 ≈ 0.7 × 0.15 = 0.11
    
    consistency_score = 0.10+0.12+0.16+0.20+0.11+0.11 = 0.80

最终决策:
  final_score = 0.395 × 1.0 × 0.80 = 0.316
  决策: **CAUTION** (25%仓位, 需人工确认)
  
  解释: 虽然第三层一致性评分尚可(0.80)，但第一层趋势确认偏弱
        (日线中性+反转已过期+周线中性)。综合评分0.316落入CAUTION区间。
        建议小仓位试探或等待更好的确认信号。
```

### 8.3 长期：完整框架（架构升级）

**目标架构**: 从当前的"4视图独立展示面板"升级为"跨周期协同决策引擎"。

```
架构变化:
  现在:  4独立线程 → 4独立渲染 → 4独立PnL
  将来:  4独立数据流 → 统一信号总线 → 跨周期过滤引擎 → 统一决策输出

新增组件:
  - SignalBus (信号总线): 收集所有周期的Schmitt信号，统一时间戳
  - CrossCycleEngine (跨周期引擎): 实现三层过滤
  - DecisionLogger (决策日志): 记录每次决策的完整三层评分和理由
  - PositionManager (仓位管理): 基于决策输出管理实际仓位
```

**关键技术点**：

1. **时间对齐**：统一 `day_offset` 基准时间，确保4周期看到相同的市场截面。
2. **周期自适应参数**：N_EWMA等参数按周期长度的平方根缩放（`N_EWMA_tf = N_EWMA_base × sqrt(tf_bar_minutes / base_bar_minutes)`）。
3. **回测验证**：对历史60分钟信号运行三层过滤，比较过滤前后的胜率和收益率。
4. **参数优化**：使用网格搜索或贝叶斯优化，在历史数据上优化各层权重和阈值。
