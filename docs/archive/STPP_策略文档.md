# STPP 策略文档 — 施密特触发器 + 预测曲线双确认交易系统

> 基于 `/filter_app/streamlit_app.py` 实际代码实现，版本 2026-06

---

## 一、策略概述

### 1.1 名称与核心思想

**STPP** (Schmitt Trigger + Parabolic Prediction) 是一个双层确认的趋势跟踪策略：

- **第一层（趋势状态）**：施密特触发器对滤波价格的速度 `v = d(filtered)/dt` 和加速度 `a = dv/dt` 施加自适应死区滞回判断，输出 `Sig_t ∈ {+1, 0, −1}`，分别表示多头、观望、空头。
- **第二层（方向预测）**：对多空切换段做曲线拟合并向前外推预测。外推段方向与信号方向一致时才入场。

两层同时确认后才开仓，离场由分段混合规则管理。

### 1.2 适用场景

- 趋势性市场（震荡市自动进入 Sig=0 观望，不交易）。
- 多周期独立配置（从 1 分钟到季线共 8 个周期）。
- 配合任意滤波算法（Savgol、Kalman、EMA 等 9 种）适应不同波动特征。

## 二、信号系统

### 2.1 施密特触发器 (`_schmitt_trigger`)

#### 2.1.1 输入量

```python
v = d(filtered_price) / dt    # 速度，滤波价一阶导数（动量）
a = d(v) / dt                  # 加速度，滤波价二阶导数（趋势加速）
```

实际计算使用 `np.gradient(filtered, t)` 得到 `v`，再对 `v` 做 `np.gradient(v, t)` 得到 `a`。

#### 2.1.2 自适应死区

死区 `ε_t` 基于速度 `v` 的 EWMA 波动率，在低波动环境下有地板保护：

```
α = 2 / (N_EWMA + 1)

μ_v[0] = v[0]
σ_v[0] = 0

for i ≥ 1:
    μ_v[i] = α · v[i] + (1 − α) · μ_v[i−1]
    σ_v[i] = √( α · (v[i] − μ_v[i])²  +  (1 − α) · (σ_v[i−1])² )
```

死区：

```
ε_t = k_ε · max(σ_t(v), σ_min)
```

| 参数 | 代码变量 | 默认值 | 范围 | 含义 |
|------|---------|--------|------|------|
| k_ε | `ke` | 0.15 | 0.01–0.50 | 灵敏度系数，越小死区越窄、信号越敏感 |
| σ_min | `sm` | 0.05 | 0.01–0.20 | 地板保护，防止低波动时 ε_t → 0 |
| N_EWMA | `ew` | 60 | 10–120 | EWMA 平滑窗口，α = 2/(N+1) |

#### 2.1.3 状态转移表

滞回逻辑：从 0 进入 ±1 需要 **双重条件**（加速度 + 速度方向同时确认）；从 ±1 退回到 0 只需 **单条件**（加速度反向突破死区）。

| 当前状态 | 条件 | 新状态 | 说明 |
|---------|------|--------|------|
| **0（观望）** | `a[i] > ε[i]` **且** `v[i] > 0` | **+1（多头）** | 正加速且正速度，双重确认入场 |
| | `a[i] < −ε[i]` **且** `v[i] < 0` | **−1（空头）** | 负加速且负速度，双重确认入场 |
| | 其他 | 0 | 继续观望 |
| **+1（多头）** | `a[i] < −ε[i]` | **0（观望）** | 明确负加速，退出多头 |
| | 其他 | +1 | 保持多头（不对 a > ε 做反应，不翻空） |
| **−1（空头）** | `a[i] > ε[i]` | **0（观望）** | 明确正加速，退出空头 |
| | 其他 | −1 | 保持空头（不对 a < −ε 做反应，不翻多） |

若 `a[i]` 或 `v[i]` 为 NaN，维持当前状态，持续时长 `dur` 递增。

#### 2.1.4 输出

```python
{
    "mu_v":    mu_v,      # v 的 EWMA 均值
    "sigma_v": sigma_v,   # v 的 EWMA 波动率
    "eps":     eps_t,     # 自适应死区序列
    "sig":     sig_t,     # 交易信号，±1/0，dtype=int
    "dur":     dur_t,     # 当前状态持续时长（含 NaN 占位）
}
```

### 2.2 多空切换对 (`_find_all_pairs`)

对 `sig_t` 做三段式处理，输出窗口内所有可交易波段：

**Step 1 — 收集非零段**：扫描 `sig_t`，提取所有连续同号区间，记录 `(start, end, val)`。

**Step 2 — 合并相邻同号段**：`+1` 与 `+1` 之间即使夹杂 `0`，也合并为一个连续多头段（含中间观望区）。同理合并空头段。

**Step 3 — 相邻异号配对**：相邻且符号不同的段，配对为 `(merged[j].start, merged[j+1].start)`——**结束于反向信号入口边缘**，而非反向段的末尾。

```python
# 示例：sig = [+1,+1,+1,0,0,-1,-1]
# Step 1 → [(0,2,+1), (5,6,-1)]
# Step 2 → 无需合并（无同号相邻）
# Step 3 → [(0, 5)]   # 多头段起点到空头段入口
```

**语义**：一个"多空对"从方向确认入场（如 +1 段起点）到反向信号入场（如 −1 段起点）为一个完整交易波段。`pair_end` 是反向信号的第一个点，即开仓位置。

### 2.3 预测曲线

#### 2.3.1 两种拟合方式

对 `filtered[pair_start : pair_end+1]`（长度 ≥ 3）做拟合。

**方式一：二次多项式 `_fit_parabolic`**

```
y = a · x² + b · x + c
# 标准最小二乘，np.polyfit(x_seg, y_seg, 2)
```

返回值：`{a, b, c, y_fit}`，其中 `y_fit` 为拟合段 y 值。

**方式二：抛物线顶点锚定 `_fit_physics_parabola`**

```
y = a · (x − x₀)² + y₀

顶点: x₀ = x[end]  (pair_end 处的 x 坐标)
      y₀ = y[end]  (pair_end 处的滤波价格，固定)
```

将顶点 `(x₀, y₀)` 锚定在对端点，仅用最小二乘求曲率 a：

```
dt  = x − x₀           ← 左半段，dt ≤ 0
dy  = y − y₀
a   = Σ(dt² · dy) / Σ(dt⁴)
```

因为 b=0、c=y₀ 已由锚定固定，拟合等价于 `y − y₀ = a · (x − x₀)²`。

返回值：`{a, b: 0.0, c: y₀, y_fit, x₀}`。

#### 2.3.2 前向外推预测

预测从 `pair_end` 向前延伸 `n_extend` 个点（默认 10）。

```python
x_pred = np.arange(pair_end, pair_end + n_extend)

# poly2 模式:
y_pred = a · x_pred² + b · x_pred + c

# parabola 模式:
y_pred = a · (x_pred − x₀)² + c   (c = y₀)
```

#### 2.3.3 预测残差

残差 = `y_pred − filtered[pair_end]`（预测值与对端点滤波价之差），在残差子图上以虚线展示。

- 红色 (`#f85149`)：预测向上（`y_pred[−1] > y_pred[0]`）
- 绿色 (`#3fb950`)：预测向下（`y_pred[−1] < y_pred[0]`）

#### 2.3.4 图表上的预测曲线

| 线段 | 颜色 | 线型 | 区间 |
|------|------|------|------|
| 拟合段 | `#f0a040` 橙色 | 实线（width=2） | `[pair_start, pair_end]` |
| 前向预测段 | `#a371f7` 紫色 | 虚线（dash） | `[pair_end, pair_end + n_extend)` |
| 预测残差 | `#f85149` 红 / `#3fb950` 绿 | 点线（dot, width=1.5） | 同上（残差子图） |

## 三、做多策略

### 3.1 入场条件（AND，全部满足）

| 条件 | 代码表达式 | 说明 |
|------|-----------|------|
| 对尾信号为多头 | `sig[pair_end] == 1` | 第二段方向确认为多头 |
| 预测方向向上 | `y_pred[−1] > y_pred[0]` | 外推末端高于起始端 |
| 对段有 ≥ 3 个数据点 | `pair_end − pair_start ≥ 3` | 拟合需要最少点数 |

入场点：`entry_idx = pair_end`，入场价：`entry_price = filtered[pair_end]`。

```python
# 代码中的精确判断
v2 = sig[pair_end]
pred_up = y_pred[-1] > y_pred[0]
is_long = (v2 == 1 and pred_up)
```

### 3.2 分段混合退出（方案 D）

退出分为两个阶段，在同一循环中按序检查：

#### 阶段一：预测保护期 `i ∈ [entry+1, entry+N_ext]`

双重保护同时生效：

**止损**（优先检查）：
```
止损触发条件: P_t < ŷ_t · (1 − s / 100)
```
其中：
- `P_t = filtered[i]`（当前滤波价）
- `ŷ_t` 为预测值（poly2: `a·i²+b·i+c`，parabola: `a·(i−x₀)²+c`）
- `s = stop_loss_pct`（止损阈值，默认 2.0）

**止盈**（每次循环均检查）：
```
止盈触发条件: sig_t[i] == −1
```

止损优先于止盈：止损条件命中即 `break`，不再继续检查止盈。

#### 阶段二：趋势跟踪期 `i ∈ [entry+N_ext+1, …]`

仅保留止盈规则，止损停用：
```
仅止盈: sig_t[i] == −1
```
让利润奔跑，不再受预测值的止损约束。

#### 边界处理

若全程未触发止损或止盈，遍历到数据末尾时 `exit_idx` 仍为 `None`，则以最后一个有效数据点离场，`exit_reason = "eod"`（end of data）。若 `entry_idx` 已是最后一个点（无后续数据可扫描），则跳过该交易。

### 3.3 收益率计算

```
做多收益率 = (exit_price − entry_price) / entry_price
```

## 四、做空策略

### 4.1 入场条件（AND，全部满足）

| 条件 | 代码表达式 | 说明 |
|------|-----------|------|
| 对尾信号为空头 | `sig[pair_end] == −1` | 第二段方向确认为空头 |
| 预测方向向下（不向上） | `not pred_up`，即 `y_pred[−1] ≤ y_pred[0]` | 外推末端不高于起始端 |
| 对段有 ≥ 3 个数据点 | `pair_end − pair_start ≥ 3` | 拟合需要最少点数 |

```python
is_short = (v2 == -1 and not pred_up)
```

入场点：`entry_idx = pair_end`，入场价：`entry_price = filtered[pair_end]`。

### 4.2 分段混合退出

与做多对称：

#### 预测保护期

```
止损触发条件: P_t > ŷ_t · (1 + s / 100)
```

实际价格超过预测值的 `(1 + s%)` 时止损。止盈条件仍为 Sig 反转：`sig_t[i] == +1`。

#### 趋势跟踪期

仅止盈：`sig_t[i] == +1`，止损停用。

### 4.3 收益率计算

```
做空收益率 = (entry_price − exit_price) / entry_price
```

## 五、PnL 双曲线架构

### 5.1 核心设计

做多（`long_pnl`）与做空（`short_pnl`）是两条**完全独立**的收益曲线，各自追踪独立的本金和交易。初始化均为 `100.0`（表示 100%）。

```python
long_pnl  = np.full(n, 100.0)   # 做多收益曲线
short_pnl = np.full(n, 100.0)   # 做空收益曲线

long_capital  = 100.0   # 做多已实现本金
short_capital = 100.0   # 做空已实现本金
```

### 5.2 逐笔填充

对每笔交易，填充对应方向的曲线：

**持仓期间** `i ∈ [entry_idx, exit_idx]`：曲线随滤波价同步波动（未实现盈亏）

```python
# —— 做多持仓期 ——
unrealized = (filtered[i] − entry_price) / entry_price
long_pnl[i] = long_capital × (1 + unrealized)

# —— 做空持仓期 ——
unrealized = (entry_price − filtered[i]) / entry_price
short_pnl[i] = short_capital × (1 + unrealized)
```

**离场后** `i ∈ [exit_idx+1, n)`：填充已实现本金值（本笔交易的最终值）

```python
# 更新已实现本金
long_capital  *= (1 + trade_return)
short_capital *= (1 + trade_return)

# 填充到数据末尾
for i in range(exit_idx + 1, n):
    long_pnl[i] = long_capital     # 或 short_pnl[i] = short_capital
```

注意：后续交易会覆盖后面区间的值。

### 5.3 Forward-fill 规则

所有交易填充完毕后，对两曲线做前向填充，消除间隙：

```python
# 做多曲线 forward-fill
last_val = 100.0
for i in range(n):
    if long_pnl[i] == 100.0 and i > 0 and last_val != 100.0:
        long_pnl[i] = last_val      # 用上一个值填充
    if long_pnl[i] != 100.0 or (i == 0):
        last_val = long_pnl[i]      # 记住当前值
```

同理对 `short_pnl`。效果：非持仓期呈水平直线（维持上一个已实现值）。

### 5.4 统计指标

在策略面板展示：

| 指标 | 计算方式 |
|------|---------|
| 交易笔数 | `len(trade_records)` |
| 胜率 | 盈利笔数（`return_pct > 0`）/ 总笔数 |
| 做多收益 | `long_pnl[−1] − 100.0`（%） |
| 做空收益 | `short_pnl[−1] − 100.0`（%） |
| 总收益 | 做多收益 + 做空收益（%） |
| 做多最大回撤 | `min((long_pnl − peak) / peak × 100)` |
| 做空最大回撤 | `min((short_pnl − peak) / peak × 100)` |

### 5.5 交易记录

每笔交易记录包含：

```python
{
    "id":           trade_id,        # 自增序号
    "type":         "long"|"short",  # 方向
    "entry_idx":    entry_idx,       # 入场索引
    "exit_idx":     exit_idx,        # 离场索引
    "entry_price":  entry_price,     # 入场滤波价
    "exit_price":   exit_price,      # 离场滤波价
    "return_pct":   trade_return*100,# 收益率(%)
    "exit_reason":  "stop_loss"|"take_profit"|"eod",
}
```

| 视图周期 | `tf` | 日线/60分钟/15分钟/5分钟 | 8 选 1 |

| 施密特显示 | `show_sch` / `sch` | True | 启用/禁用 Schmitt + 衍生行 |
| 时间偏移 | `_day_offset` | 0 | 按天前移/后移历史窗口 |

### 7.5 滤波参数

9 种滤波器各有独立参数，详见 `FILTERS` 字典（代码行 138–202）。以默认 savgol 为例：

| 滤波参数 | 范围 | 默认值 |
|---------|------|--------|
| window（窗口大小） | 5–101 | 21 |
| order（多项式阶数） | 1–5 | 2 |

## 八、配置导入导出

### 8.1 JSON 格式

导出为 `filter_config.json`，包含全局参数和 4 个视图（v0–v3）的全部参数。

### 8.2 Key 命名规则

**全局参数**：

| Key | 含义 |
|-----|------|
| `market` | 市场（美股/A股/港股） |
| `ticker` | 股票代码 |
| `global_f` | 滤波器 ID（如 `"savgol"`） |
| `global_dual` | 是否双滤波对比 |
| `global_f2` | 滤波器 2 ID |

**视图参数** `v{i}`（i = 0,1,2,3）：

| Key 模式 | 含义 |
|---------|------|
| `v{i}_tf` | 周期 |
| `v{i}_n` | 数据点数 |
| `v{i}_sch` | 施密特开关 |
| `v{i}_pred` | 预测开关 |
| `v{i}_ke` | k_ε |
| `v{i}_sm` | σ_min |
| `v{i}_ew` | N_EWMA |
| `v{i}_fm` | 拟合方式（`"poly2"` 或 `"parabola"`） |
| `v{i}_next` | 预测点数 |
| `v{i}_fc` | 滤波 1 颜色 |
| `v{i}_fc2` | 滤波 2 颜色 |
| `v{i}_strat` | 策略开关 |
| `v{i}_sl` | 止损阈值 |

**滤波参数** Key = `"{label}_v{i}_f1_{filter_id}"` 或 `"{label}_v{i}_f2_{filter_id}"`，其中 `label` 为滤波参数的中文标签名（如 `"窗口大小"`、`"多项式阶数"`）。

### 8.3 兼容性

导入时使用 MD5 哈希检测重复文件（相同文件不重复导入）。导出配置使用 `.get()` 兜底默认值，旧版不含新参数的配置可直接导入，缺失参数自动填充默认值。
