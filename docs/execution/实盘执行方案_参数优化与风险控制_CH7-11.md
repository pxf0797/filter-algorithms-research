## 第7章 参数优化体系

### 7.1 3690_HK 配置诊断

当前配置（`config/3690_HK.json`）的核心参数在四个周期（15分钟/60分钟/日线/周线）上完全一致：

| 参数 | 值 | 问题诊断 |
|------|-----|---------|
| Savgol window | 13 (所有周期) | 周线120个bar，窗口13对周线过小（仅覆盖约3个月）；对15分钟过大（覆盖约3小时交易），平滑过度丢失短期信号 |
| Savgol order | 4 (所有周期) | 在高窗口下可能过拟合噪声，周线数据点稀疏时尤其严重 |
| k_eps | 0.15 (所有周期) | 15分钟高频噪声大需要更高k_eps抑制假信号；周线趋势明确本可更低（0.10-0.12）以捕捉早期信号 |
| sigma_min | 0.05 (所有周期) | 周线波动率在0.3-0.8之间，地板保护基本不起作用；15分钟波动率0.02-0.05，地板保护频繁触发 |
| EWMA | 60 (所有周期) | 15分钟约60个bar=15小时波动率估计，对大周期正常；周线60bar约15个月，反应极慢 |
| 固定止损 | 2.0% (所有周期) | 见7.3节详细分析 |

**核心发现（来源：T3研究成果）**：
- SG window 参数敏感度最高，四周期共享同一 window 意味着各周期均非最优
- 2%固定止损在40-50%年化波动率（美团历史）下，大约有15-20%的概率在正常价格波动中被突破——不符合止损"只拦灾难性亏损"的初衷

### 7.2 参数敏感度矩阵

以下敏感度排序基于T3参数扫描研究，从高到低排列：

| 优先级 | 参数 | 敏感度 | 原因 |
|--------|------|--------|------|
| P0 | SG window | 极高 | 控制滤波滞后 vs 平滑的权衡，直接影响梯度计算 |
| P1 | 止损方式 | 高 | 固定止损 vs ATR动态止损对PnL影响可达3-5倍 |
| P2 | k_eps | 高 | 直接控制信号频率，±0.05可导致信号数翻倍或减半 |
| P3 | 四周期权重 | 中-高 | 加权方式决定多周期信号合成的判决阈值 |
| P4 | sigma_min | 中 | 仅在低波动期生效，对中高波动标的边际影响有限 |
| P5 | EWMA span | 中 | 影响波动率自适应速度，但k_eps+sigma_min覆盖大部分功能 |

### 7.3 ATR动态止损替换方案

**问题定量分析**：

美团(3690.HK)近一年ATR数据：
- 日线ATR(14): 约4-8 HKD（当前价格约150 HKD，即2.7-5.3%）
- 15分钟ATR(14): 约0.8-2.0 HKD（0.5-1.3%）

2%固定止损意味着：当天线ATR=6 HKD时，正常日内波动即可能触及止损。在波动率放大期（如财报前后），ATR可达10+ HKD，2%止损必然被频繁触发。

**推荐方案**：ATR动态止损

| 周期 | ATR乘数 | 对应波动率覆盖 | 预期被触发概率 |
|------|---------|---------------|---------------|
| 15分钟 | 1.5-2.0x ATR(10) | 覆盖85-95%价格波动 | 约5-8%（仅尾部风险） |
| 60分钟 | 1.8-2.5x ATR(10) | 覆盖88-97%价格波动 | 约3-6% |
| 日线 | 2.0-3.0x ATR(14) | 覆盖90-98%价格波动 | 约2-5% |
| 周线 | 2.5-3.5x ATR(14) | 覆盖92-99%价格波动 | 约1-3% |

**实现方式**（伪代码）：

```python
def atr_stop_loss(entry_price, atr_series, multiplier, atr_window):
    """
    返回动态止损价。
    做多: entry_price - atr * multiplier
    做空: entry_price + atr * multiplier
    """
    current_atr = atr_series.iloc[-1]
    return entry_price - current_atr * multiplier  # for long
```

**关键设计决策**：
- ATR基于入场时刻的值冻结，不随持仓期重新计算（否则止损线持续漂移，失去意义）
- 保护期内使用ATR止损替代固定比例止损
- 趋势跟踪期切换回Schmitt信号反转止盈，同时保持ATR止损作为保底

### 7.4 三阶段调优工作流（3 Phase）

**Phase 1: 敏感度扫描**（约1天）

目标：确定各参数对PnL的边际影响，缩小优化空间。

```
步骤：
1. 固定其他参数基线（使用当前配置值）
2. 对每个参数独立做网格扫描
   - SG window: [5, 7, 9, 11, 13, 15, 17, 21, 25]
   - k_eps: [0.05, 0.10, 0.15, 0.20, 0.25, 0.30]
   - sigma_min: [0.02, 0.03, 0.05, 0.08, 0.10]
   - EWMA: [20, 30, 40, 60, 80, 100]
3. 记录每组参数的 Sharpe、交易次数、最大回撤
4. 识别各参数的"平台区"——参数变化但性能相对稳定的区域
```

验证指标：
- 单参数扫描的Sharpe变化曲线（应呈单峰或平台，而非锯齿）
- 交易次数应在可接受范围（15分钟周期≥50笔/年，日线周期≥15笔/年）

**Phase 2: 多周期联合优化**（约3-5天）

目标：各周期独立参数 + 加权信号合成的联合调优。

```
步骤：
1. 四周期解耦参数：每个周期独立扫描SG window
   - 15分钟: window=9-15  (短窗口，适应高频)
   - 60分钟: window=11-17
   - 日线: window=13-21
   - 周线: window=7-13   (周线数据点稀疏，适中小窗口)
2. 四周期独立k_eps扫描
   - 原则：周期越短，k_eps越大
   - 15分钟: 0.18-0.25
   - 60分钟: 0.12-0.20
   - 日线: 0.10-0.18
   - 周线: 0.08-0.15
3. 信号加权合成调优
   - 推荐基线权重：周:日:60m:15m = 4:3:2:1（来源：T3研究成果）
   - 权重敏感度测试：各调高/调低1，观察PnL稳定性
4. ATR乘数联合调优
   - 短周期乘数偏小（1.5-2.0），长周期偏大（2.0-3.0）
```

验证指标：
- 联合参数组合的滚动Sharpe稳定性（4周滚动窗口，标准差<0.3）
- 各周期贡献度：移除任一周期后PnL下降是否>15%
- 参数平台区确认：最优参数±10%范围内Sharpe变动<10%

**Phase 3: 稳健性验证**（约2天）

目标：确认参数组合在不同市场环境下表现稳定。

```
1. Walk-Forward Analysis (WFA):
   - 训练窗口: 6个月
   - 验证窗口: 3个月
   - 步进: 1个月
   - 要求: 80%以上的WFA窗口Sharpe>0

2. 年化滚动测试:
   - 按年分割数据（2021、2022、2023、2024、2025）
   - 跨年参数稳定性检查

3. Monte Carlo 残差扰动:
   - 对残差序列进行Bootstrap重抽样
   - 生成1000条扰动价格路径
   - 验证90%置信区间内PnL>0

4. 体制转移测试:
   - 高波动期（VIX>25等效）vs 低波动期
   - 上涨趋势 vs 下跌趋势 vs 震荡
   - 策略应在2/3以上的体制分类中表现正向
```

验证指标：
- WFA的PBO(Probability of Backtest Overfitting) < 0.3
- 不同年份的Sharpe最大值/最小值比值 < 2.0
- Monte Carlo试验中PnL>0的概率 > 70%

### 8.3 信号重绘问题的根因分析

**问题定位**：

`filter_app/streamlit_app.py`第1558行（原单文件版本，模块化后移至 `filter_app/services/filter_engine.py`，行为相同）：
```python
_v = np.gradient(filtered, t)
_a = np.gradient(_v, t)
```

`np.gradient`的默认行为是中心差分：
```python
gradient[i] = (x_{i+1} - x_{i-1}) / 2
```

这意味着：
- **梯度值依赖于前后各1个点的数据**
- 在bar未关闭时计算梯度，下一个bar的数据到来会改变当前bar的梯度值
- Schmitt Trigger的入场/离场判定依赖于梯度（`a[i] > eps_t[i] and v[i] > 0`），梯度值的重绘会直接导致信号的重绘

**具体影响路径**（以多头为例）：

```
新bar数据到来
  → 中心差分重新计算历史梯度
    → 历史加速度a值变化
      → 导致Schmitt Trigger入场/离场边界偏移
        → 信号重绘（已交易的历史bar的sig_t值改变）
          → PnL回溯计算变化（回测不可复现）
```

**修正方案："冻结+仅已关闭bar"**

方案A — 保守（推荐用于MVP）：

```python
def compute_gradient_safe(filtered, t):
    """仅使用已关闭bar的数据计算梯度。
    方法：将序列整体shift(1)，使当前bar不参与计算。
    """
    # shift(1)使当前bar的梯度基于过去已确定的数据
    filtered_shifted = np.concatenate(([filtered[0]], filtered[:-1]))
    v = np.gradient(filtered_shifted, t)
    a = np.gradient(v, t)
    # 返回时shift回来（最后一个值为NaN，因为无新数据）
    return v, a
```

方案B — 严格（推荐用于生产级）：

```python
class BarBuffer:
    """延迟一个bar的数据缓冲区，确保所有计算基于已关闭bar。"""
    def __init__(self, maxlen=300):
        self.buffer = deque(maxlen=maxlen)
    
    def append(self, close_price, t):
        """追加当前bar数据后，从已关闭bar缓冲区计算梯度。"""
        self.buffer.append(close_price)
        if len(self.buffer) < 3:
            return None, None
        # 转换为numpy array，取除最后一个元素外的所有已关闭bar
        closed = np.array(list(self.buffer))[:-1]
        closed_t = np.arange(len(closed))
        v = np.gradient(closed, closed_t)
        a = np.gradient(v, closed_t)
        return v[-1], a[-1]  # 仅返回最后一个已关闭bar的梯度
```

**验证方法**：
在Bar Replay模式下逐步播放历史数据，每个bar关闭后记录一次梯度值和信号值。播放完毕后，对比三次计算的梯度是否完全一致。若不一致则说明存在重绘。

### 8.4 前向预测的实盘管理

**问题**：8bar前向预测的抛物线在实盘中不断修正。回测中看到的"完美"连续预测曲线，在实盘中会跟随新数据不断漂移。

**解决方案——"冻结"机制**：

```python
class PredictionFreezer:
    """
    前向预测冻结管理器。
    
    核心规则:
    - 每个bar关闭时，"冻结"该bar上的预测值到永久存储
    - 后续不修改已冻结的预测值
    - 当前进行中的预测仅用于止损参考线，不用于入场信号
    """
    
    def __init__(self, max_frozen=1000):
        self.frozen = {}  # bar_idx → frozen_prediction_value
    
    def freeze_bar(self, bar_idx, prediction_value):
        """bar关闭时调用：冻结该bar的预测值。"""
        if bar_idx not in self.frozen:  # 首次写入即冻结
            self.frozen[bar_idx] = prediction_value
    
    def get_reference(self, bar_idx):
        """获取已冻结的参考值，用于止损计算。"""
        return self.frozen.get(bar_idx, None)
    
    def is_frozen(self, bar_idx):
        return bar_idx in self.frozen
```

**实盘中的实际行为**：
- 新预测曲线的历史部分不断被覆盖，但已冻结的值保持不变
- 止损参考线呈现"逐步冻结石"形态——第一部分来自上一个bar的预测，第二部分来自上上个bar的预测，以此类推
- 这与回测中看到的连续曲线不同，是实盘的正常现象

**限制**：冻结后的预测曲线在bar关闭前用于止损参考，但不可用于入场信号。入场信号仅基于**已关闭bar的Schmitt Trigger状态**。

## 第9章 风险控制体系

### 9.1 四层级风控矩阵

风控系统分层设计，每层有明确的量化阈值和独立执行路径（来源：T4研究成果）。

| 层级 | 名称 | 监控指标 | 阈值 | 触发动作 | 恢复条件 |
|------|------|---------|------|---------|---------|
| L1 | 单笔风控 | 单笔亏损占账户比例 | 0.5-1.5% | 市价止损单 | 自动恢复（下次交易） |
| L1 | 单笔风控 | ATR止损突破 | 1.5-3.0x ATR(10) | 市价止损单 | 自动恢复 |
| L1 | 单笔风控 | 保护期内最大持有bar | n_extend+3 bar | 市价平仓 | 自动恢复 |
| L2 | 日内风控 | 日内累计亏损（含未实现） | 账户的3-5% | 暂停当日开仓 | 次日自动恢复 |
| L2 | 日内风控 | 日内最大交易次数 | 策略频率×1.5倍 | 暂停入场 | 次日自动恢复 |
| L2 | 日内风控 | 日内最大换手率 | 账户资金的50% | 暂停入场 | 次日自动恢复 |
| L3 | 组合风控 | 峰值回撤 | 8-12% | 全部平仓+暂停 | 人工审核后重启 |
| L3 | 组合风控 | 连续亏损交易次数 | 5-8笔 | 暂停+报警 | 人工审核 |
| L3 | 组合风控 | 波动率比率（当前ATR/基准ATR） | >2.0x | 仓位减半 | 比率回落后恢复 |
| L3 | 组合风控 | 4周期信号分歧度 | >60%信号不一致 | 暂停入场 | 分歧度回落 |
| L4 | 系统风控 | 数据断连时长 | >60秒 | 暂停交易+报警 | 连接恢复+数据校验 |
| L4 | 系统风控 | 异常订单率（撤/改/拒占比） | >20% | 暂停+系统检查 | 人工确认 |
| L4 | 系统风控 | 手动Kill Switch | 人工触发 | 全部平仓+暂停 | 人工手动重启 |

### 9.2 熔断器设计

熔断器是风控体系的核心执行组件，需独立于策略逻辑运行。

**状态机**：

```
        +----------+
        |  正常运作  | <────── 人工确认恢复
        +----+-----+               ↑
             | 触发任一L3/L4条件      |
             v                       |
        +----------+           +----+------+
        |  触发熔断  | ──冷却期 ──>  冷却阶段  |
        |  (暂停开仓) |  (N分钟)   | (仍持仓)  |
        +----------+           +-----------+
             |                        |
             | 达到最大回撤阈值         | 人工审核
             v                        v
        +----------+           +----------+
        |  紧急平仓  |           | 人工恢复  |
        |  (全部平仓) |           | 手动重启  |
        +----------+           +----------+
```

**关键规则**（来源：T4研究成果）：
1. 熔断器独立于策略信号逻辑运行，**不能被策略绕过**
2. 熔断后**不自动恢复**，需人工审核后手动重启
3. 冷却期不允许开新仓，但允许关闭现有头寸
4. 所有触发事件记录时间戳、触发原因、当前仓位、资产净值到不可变日志
5. 连续熔断（24小时内触发≥3次）自动降级至紧急平仓状态

**熔断器接口定义**：

```python
class CircuitBreaker:
    """独立熔断器组件，不与策略共享任何状态。"""
    
    STATE_NORMAL = "NORMAL"
    STATE_COOLDOWN = "COOLDOWN"   # 冷却中（可平仓不可开仓）
    STATE_LIQUIDATE = "LIQUIDATE" # 紧急平仓
    STATE_HALT = "HALT"           # 人工恢复中
    
    def __init__(self, max_drawdown=0.10, consecutive_losses=6):
        self.state = self.STATE_NORMAL
        self.peak_capital = 0.0
        self.consecutive_losses = 0
        self.drawdown = 0.0
        self.event_log = []
    
    def check(self, current_capital):
        """
        检查是否触发熔断。
        返回 (should_trigger: bool, new_state: str)
        """
        # 计算当前回撤
        self.peak_capital = max(self.peak_capital, current_capital)
        self.drawdown = (self.peak_capital - current_capital) / self.peak_capital
        
        # L3 检查
        if self.drawdown >= max_drawdown:
            return True, self.STATE_LIQUIDATE
        if self.consecutive_losses >= consecutive_losses:
            return True, self.STATE_COOLDOWN
        
        return False, self.STATE_NORMAL
    
    def record_trade(self, pnl_pct):
        """记录单笔交易结果到熔断器。"""
        if pnl_pct < 0:
            self.consecutive_losses += 1
        else:
            self.consecutive_losses = 0
        self.event_log.append({"time": now(), "pnl": pnl_pct})
    
    def reset(self):
        """人工重置熔断器。"""
        self.state = self.STATE_NORMAL
        self.consecutive_losses = 0
```

### 9.3 跨周期风险叠加管理

四周期同时持仓时，风险不是简单的线性叠加。需管理以下叠加效应：

1. **仓位叠加**：同向信号在多个周期同时开仓造成单方向过度暴露
   - 解决方案：设置单方向最大总仓位（账户的30%），四周期合并计算

2. **止损冲突**：15分钟止损在日线趋势中频繁触发，造成"被震下车"
   - 解决方案：更低周期（15分钟/60分钟）在更高周期持仓期间放宽止损乘数1.5倍

3. **信号矛盾**：周线做多，15分钟做空时的仓位管理
   - 规则：长周期信号优先——持仓方向与周线信号一致，短周期反向信号仅允许减仓，不允许反向开仓

```python
def compute_net_position(higher_signal, lower_signal, higher_weight=0.4):
    """
    跨周期仓位计算。
    长周期信号决定方向，短周期信号决定加减仓。
    """
    if higher_signal == 0:
        return lower_signal  # 长周期无信号，跟随短周期
    
    if higher_signal == lower_signal:
        return higher_signal  # 同向，全仓
    
    # 长周期信号 vs 短周期反向：允许减仓
    return higher_signal * 0.5  # 保留50%仓位
```

### 9.4 港股市场特殊风险

港股市场的独特性对策略有直接影响（来源：T4研究成果）：

| 港股特性 | 影响 | 应对措施 |
|---------|------|---------|
| **T+0交易** | 可日内多次进出，但需控制交易成本 | 日内同一方向最多3次交易 |
| **T+2结算** | 卖出后资金T+2到账，但T+0可用再买 | 预留5-10%结算准备金 |
| **无涨跌停保护** | 单日可跌50%+，极端行情无缓冲 | ATR止损乘数偏保守（1.5-2.0x），熔断器阈值下调至8% |
| **0.13%印花税** | 买卖双向，每100万HKD交易=2600HKD税费 | 计入交易成本模型，短周期预期收益需>0.5%/笔才可覆盖 |
| **开收市竞价时段** | 09:00-09:20和16:00-16:10成交价不可控 | 策略信号逻辑避开竞价时段，仅在连续竞价时段交易（09:30-16:00） |
| **最小变动价位** | 不同价格区间tick size不同 | 止损单使用市价单而非限价单（避免因tick size导致无法成交） |
| **台风/黑雨停市** | 不定时停市，可能导致策略持仓无法操作 | 引入交易日历事件处理逻辑，提前平仓或设宽止损 |
| **汇率风险** | 人民币→港币换算，汇率波动影响实际收益 | 账户货币统一为港币，汇率损益单独监控 |

**特别警示**：
- 港股部分中小盘股流动性极差，大单进出可能导致剧烈滑点
- 策略标的3690.HK（美团）是恒生科技成分股，流动性充足，但仍需监控盘中瞬时大单
- 港股财报季（3月/8月）价格跳空频繁，建议在此期间手动降低仓位至50%

### 11.2 策略漂移检测

策略漂移是实盘中最容易被忽视的风险——策略可能在无亏损的情况下逐渐失效。

**检测方法**：

1. **滚动窗口绩效对比**（每30笔交易触发）
   - 计算滚动30笔的Sharpe、胜率、平均盈亏比
   - 与回测期全体指标对比
   - 告警阈值：Sharpe偏差>50%，胜率偏差>20%

2. **KS检验**（每日收盘后触发）
   - 对实盘和回测的每日PnL分布做Kolmogorov-Smirnov检验
   - p<0.05时告警——说明实盘收益分布已显著偏离回测
   - 注意：样本量<30笔时KS检验效力不足

3. **PSI (Population Stability Index)**（每周触发）
   - 监控Schmitt Trigger信号值分布的变化
   - PSI>0.1提示警告，>0.25触发熔断
   - 计算方式：
     ```
     PSI = Σ(实盘占比_i - 回测占比_i) * ln(实盘占比_i / 回测占比_i)
     ```

4. **交易频率监控**（每日触发）
   - 跟踪当日交易次数vs回测日均交易次数
   - 偏差>30%告警（可能说明市场体制变化）

5. **持仓时长分布**（每周触发）
   - 跟踪各周期的平均持仓时长
   - 与回测显著偏差可能说明策略行为改变

### 11.3 异常交易自动熔断

| 异常场景 | 检测方式 | 熔断动作 | 恢复条件 |
|---------|---------|---------|---------|
| 连续亏损≥N笔 | 熔断器L3 | 暂停策略，报警 | 人工审核 |
| 日内亏损超限 | L2风控 | 暂停当日开仓 | 次日自动 |
| 数据断连>60秒 | 数据监控 | 暂停交易，报警 | 连接恢复+校验 |
| 信号异常密集 | 交易频率监控 | 暂停入场 | 人工审核 |
| 滑点显著扩大 | 执行监控 | 报警并调整订单类型 | 人工确认 |
| 券商API异常 | 异常订单率 | 系统暂停 | 人工确认 |
| 跨周期状态不一致 | 状态管理 | 暂停交易 | 自动恢复（至持续一致） |
