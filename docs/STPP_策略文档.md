# 基于施密特触发器与抛物线预测的量化交易策略

## 一、策略概述

### 1.1 策略名称
**STPP 策略** (Schmitt Trigger + Parabolic Prediction)

### 1.2 核心思想
利用**施密特触发器(Schmitt Trigger)**产生的自适应趋势信号(Sig_t)与**抛物线预测曲线**的前向方向，构建双重确认的交易系统：

- **第一层确认（趋势状态）**：施密特触发器基于价格滤波后的速度(v)和加速度(a)，通过自适应死区(epsilon_t)判断当前市场处于多头(+1)、空头(-1)还是观望(0)状态。滞回机制防止频繁切换。
- **第二层确认（方向预测）**：对每个多空切换段做抛物线拟合，前向延伸产生预测曲线。预测段向上(y_ext呈上升趋势)确认多头，向下确认空头。
- **离场规则**：以预测残差突破阈值作为止损，以Sig信号翻转为确认止盈。

### 1.3 适用场景
- 适用于**趋势性市场**，在震荡市中会自动进入观望状态(0)，规避无方向行情。
- 多周期适用（日线、60分钟、15分钟、5分钟等），不同周期可独立配置参数。
- 配合滤波算法(Savgol、Kalman、EMA等)可适应不同波动特征的标的。

## 二、信号系统说明

### 2.1 施密特触发器 (`_schmitt_trigger`)

#### 2.1.1 输入量
```python
v = d(filtered_price) / dt   # 速度：滤波价格的一阶导数，代表趋势动量
a = d(v) / dt                # 加速度：滤波价格的二阶导数，代表趋势加速
```

#### 2.1.2 自适应死区计算
```python
sigma_v[t] = EWMA_volatility(v[t])           # EWMA波动率
epsilon_t  = k_epsilon * max(sigma_v[t], sigma_min)  # 自适应死区
```
- `k_epsilon`（默认0.15）：灵敏度系数，越小越敏感，越小死区越窄
- `sigma_min`（默认0.05）：地板保护，防止低波动环境下死区收缩至零
- `N_EWMA`（默认60）：EWMA平滑周期，alpha = 2/(N+1)，越大越平滑

#### 2.1.3 状态转移规则（带滞回）

| 当前状态 | 条件 | 新状态 | 含义 |
|---------|------|--------|------|
| 0 (观望) | a > ε AND v > 0 | +1 | 多头入场：正加速+正速度 |
| 0 (观望) | a < -ε AND v < 0 | -1 | 空头入场：负加速+负速度 |
| 0 (观望) | 其他 | 0 | 继续观望 |
| +1 (多头) | a < -ε | 0 | 多头离场：明确负加速 |
| +1 (多头) | 其他 | +1 | 保持多头 |
| -1 (空头) | a > ε | 0 | 空头离场：明确正加速 |
| -1 (空头) | 其他 | -1 | 保持空头 |

关键设计要点：
- **从0到±1需要双重条件同时满足**（速度方向+加速度突破死区），入场严格
- **从±1到0只需单条件**（加速度反向突破死区），离场相对宽松
- **滞回效应**：多头中不对a>ε做反应（不会翻空），空头中不对a<-ε做反应（不会翻多），避免在死区边缘频繁抖动

#### 2.1.4 输出
```python
{
    "sigma_v": sigma_v,   # v的EWMA波动率
    "eps":     epsilon_t, # 自适应死区序列
    "sig":     sig_t,     # 交易信号 ±1/0
    "dur":     dur_t,     # 当前状态持续时长
}
```

### 2.2 多空切换对 (`_find_all_pairs`)

#### 2.2.1 处理流程
```
Step 1: 扫描 sig_t，提取所有非零连续段 → [(start, end, val), ...]
Step 2: 合并相邻同号段（+1与+1之间的0区域并入前一段）
Step 3: 相邻异号段配对 → [(long_start, short_entry), ...]
```

#### 2.2.2 语义
- 一个"多空对"从信号确认入场（如+1开始）到反向信号确认入场（如-1开始）为一整个交易波段
- 对尾 end = 反向信号的**入口边缘**（即翻空前最后一个+1点），而非反向信号结束点
- 合并规则确保中途的短暂观望(0)不会断开趋势连续性

### 2.3 预测曲线

#### 2.3.1 拟合方式

**方式一：二次多项式拟合 (`_fit_parabolic`)**
```python
y = a·x² + b·x + c
# 对 [pair_start, pair_end] 区间做最小二乘拟合
```

**方式二：抛物线顶点锚定拟合 (`_fit_physics_parabola`)**
```python
y = a·(x - x₀)² + y₀
# 顶点 (x₀, y₀) = (x[pair_end], filtered[pair_end]) 固定
# 仅拟合曲率参数 a，保证曲线在端点自然衔接
```

#### 2.3.2 前向预测
- 拟合段：橙色实线(#f0a040)，覆盖整个多空对区间 `[pair_start, pair_end]`
- 预测段：紫色虚线(#a371f7)，从 pair_end 向前延伸 `n_extend` 个点
- 预测残差：预测值与终点滤波价之差，在残差子图上展示。红色(#f85149)=预测向上，绿色(#3fb950)=预测向下

## 三、做多策略(Long)

### 3.1 入场条件（AND逻辑，全部满足才入场）

| 序号 | 条件 | 代码逻辑 | 说明 |
|------|------|---------|------|
| 1 | 预测方向向上 | `y_ext[-1] > y_ext[0]` | 抛物线前向预测段末端高于起始端，确认上行趋势 |
| 2 | 多头Sig段 | `sig[i] == +1` | 施密特触发器确认当前处于多头状态 |
| 3 | 对段长度足够 | `pair_end - pair_start >= 3` | 至少3个数据点才能做有效拟合 |

> 入场价格 = 满足条件时对应的`filtered[i]`（滤波价格）。入场时机为预测方向确认且Sig为+1的第一个时间点。

### 3.2 止损条件

```
止损触发条件：filtered[t] < predicted[t] * (1 - stop_loss_pct)
```

即实际滤波价格跌破预测价格的 `stop_loss_pct` 百分比时止损离场。

**公式说明**：
- `predicted[t]`：根据拟合参数计算的前向预测值
- `stop_loss_pct`：可调止损阈值，默认2%（范围0.5%-10%）
- 含义：市场实际走势弱于预测超过阈值，预测失效，强制离场

### 3.3 止盈条件

```
止盈触发条件：sig[t] 从 +1 翻转为 0 或 -1
```

施密特触发器确认多头趋势耗尽（加速反向突破死区），以信号翻转为确认离场。

### 3.4 退出优先级
1. 止损（价格保护，优先检查）
2. 止盈（信号确认，次优先）
3. 如无止损也无止盈，持仓至对段自然结束

## 四、做空策略(Short)

### 4.1 入场条件（AND逻辑）

| 序号 | 条件 | 代码逻辑 | 说明 |
|------|------|---------|------|
| 1 | 预测方向向下 | `y_ext[-1] < y_ext[0]` | 前向预测段末端低于起始端 |
| 2 | 空头Sig段 | `sig[i] == -1` | 确认当前处于空头状态 |
| 3 | 对段长度足够 | `pair_end - pair_start >= 3` | 至少3个数据点 |

### 4.2 止损条件

```
止损触发条件：filtered[t] > predicted[t] * (1 + stop_loss_pct)
```

做空时，实际价格高于预测超过阈值，止损离场。

### 4.3 止盈条件

```
止盈触发条件：sig[t] 从 -1 翻转为 0 或 +1
```

### 4.4 退出优先级

1. 止损（优先）
2. 止盈（次优先）
3. 自然结束

## 五、PnL计算公式

### 5.1 单笔收益率

```python
# 做多
long_return = (exit_price - entry_price) / entry_price

# 做空
short_return = (entry_price - exit_price) / entry_price
```

其中：
- `entry_price = filtered[entry_index]`
- `exit_price = filtered[exit_index]`
- `exit_index` = min(止损触发索引, 止盈触发索引, 对段结束索引)

### 5.2 累积收益（初始本金100%）

```python
cumulative_pnl = [100.0]  # 初始本金
for each trade in trade_records:
    cumulative_pnl.append(cumulative_pnl[-1] * (1 + trade.return_pct))
```

### 5.3 逐点PnL曲线（与价格时间轴对齐）

```python
pnl_curve[t] = 100.0  # 初始值（在所有交易之前）
for each trade:
    entry_t = trade.entry_idx
    exit_t = trade.exit_idx
    for t in range(entry_t, exit_t + 1):
        current_price = filtered[t]
        if trade.is_long:
            unrealized_return = (current_price - trade.entry_price) / trade.entry_price
        else:
            unrealized_return = (trade.entry_price - current_price) / trade.entry_price
        pnl_curve[t] = 100.0 * (1 + unrealized_return)
```

### 5.4 关键统计指标

在策略面板中展示：
- **总交易笔数**：符合入场条件的多空对数量
- **胜率**：盈利交易 / 总交易
- **累积收益率**：最终PnL - 100%
- **最大回撤**：从峰值到谷底的最大跌幅
- **平均单笔收益**：所有交易的平均return_pct
- **盈亏比**：平均盈利 / 平均亏损（绝对值）

1. **数据不足处理**：当 `pred_pairs` 为空或对段长度不足时，`_compute_strategy_pnl` 返回空结果，图表降级
2. **颜色一致性**：新增PnL颜色与现有Sig_t子图一致（绿多红空）

5. **导出兼容性**：旧版配置不含新参数，使用 `.get()` 兜底默认值
