# 回测模式设计文档审查报告

**审查日期**：2026-07-02
**审查范围**：`backtest-final-design.md`（重点是第 1 章和第 4 章）vs `backtest-ui-mockup.html`
**审查基准**：前轮对话核心结论（回测展示 ≡ 浏览展示，差异仅在控制条 + bar_index 导航）

---

## 总体结论：需要更新

设计文档整体质量高，与 UI 效果图**大方向一致**，但存在 **3 处不一致**和 **2 处遗漏**需要修正。所有问题集中在细节层面，不影响核心架构。

### b. §4.2 控制条规格 — 按钮/速度/slider 完整性

**结论**：⚠️ 基本通过，有 2 处小问题

| 检查项 | 设计文档 | UI 效果图 | 状态 |
|--------|---------|-----------|------|
| 按钮数量 | 5 个（⏮ ◀ ▶/⏸ ▶▶ ⏭） | 5 个（⏮ ◀ ⏸ ▶▶ ⏭） | ✅ |
| 速度选项 | 6 档（0.25x~10x） | 6 档（0.25x~10x） | ✅ |
| 进度 slider | range(0, total_bars-1), key=`_bt_progress_slider` | range slider + 日期标签 | ✅ |
| 空数据守卫 | disabled 按钮 + warning | （效果图未展示此状态，但属于边界场景） | ✅ |
| 按钮分组 | 文档未描述分组 | 效果图用竖线分隔：⏮ \| ◀ ⏸ ▶▶ \| ⏭ | ⚠️ 见问题 1 |
| 播放按钮状态切换 | 文档只说图标 ▶ / ⏸，widget 为 `st.button` | 效果图展示 ⏸（暂停态），绿色背景 | ⚠️ 见问题 2 |

**问题 1**（轻微）：§4.2.1 按钮规格表未描述按钮间的视觉分组（分割线）。效果图使用 `|` 将按钮分为三组：`⏮` | `◀ ⏸ ▶▶` | `⏭`。建议在规格表中补充"分组"列，或在布局描述中加入分隔线说明。

**问题 2**（轻微）：§4.2.1 列 3（播放/暂停）的 Widget 写为 `st.button`，但实际 Streamlit 中需要根据 `_is_playing` 状态动态切换按钮文案和图标。文档应补充说明："按钮文案/图标根据 `_is_playing` 动态切换：False 时显示 ▶，True 时显示 ⏸"。效果图中播放按钮有独立样式（绿色背景 `.play-btn`），文档未提及此样式差异。

### c. §4.3 状态显示 — 格式正确性

**结论**：⚠️ 格式不一致

| 检查项 | 设计文档 (§4.3.1) | UI 效果图 | 状态 |
|--------|-------------------|-----------|------|
| 显示内容 | `📍 bar {N}/{T} \| {date}` | `📍 bar 427/1250` + `📅 2025-09-15 14:30` | ⚠️ 格式不同 |
| 播放状态 | 同一行追加 `\| ▶ 播放中`（绿色 markdown） | 独立一行显示 `▶ 播放中`（绿色） | ⚠️ 格式不同 |
| 空状态 | `_cb_mode=False` 时不渲染 | 浏览模式面板无此区域 | ✅ |
| bar 编号 | `bar_index + 1`（1-indexed 显示） | `427`（= 426+1，1-indexed） | ✅ |

**问题 3**：设计文档使用单行 pipe 分隔格式：
```
📍 bar {bar_index + 1}/{total_bars} | {formatted_date} | ▶ 播放中
```
效果图使用多行格式：
```
📍 bar 427/1250
📅 2025-09-15 14:30
▶ 播放中
```
需要决定最终格式。建议统一为效果图的多行格式（可读性更好），并更新 §4.3.1 的描述。

### d. §4.4 播放循环 — 逻辑正确性

**结论**：✅ 通过，逻辑正确

- 入口守卫：`_cb_mode=True` AND `_is_playing=True` ✅
- 到达末尾自动暂停：`bar_index >= total_bars - 1` ✅
- 速度映射正确：`delay = speed_map[speed_label]` → `time.sleep(delay)` ✅
- slider 拖拽暂停：on_change 回调中设 `_is_playing = False` ✅
- 速度切换不中断播放：仅写状态，下个 cycle 生效 ✅
- 纯 Streamlit 驱动，不依赖线程 ✅

**结论**：⚠️ 缺少控制条维度

效果图总结表有 10 个维度，其中以下维度在设计文档 §1.2 中**缺失**：

## 遗漏项

### 遗漏 1：`_play_speed_label` 状态键未声明

**位置**：§2.4.1 新增 session_state 键清单

§2.4.1 声明了 `_play_speed`（float, 默认 0.5）但未声明 `_play_speed_label`。然而 §4.2.1 列 6 和 §4.4.2 步骤 4 均引用了 `_play_speed_label`。

**建议**：在 §2.4.1 表中增加一行：
```
| `_play_speed_label` | `str` | `"1x"` | 播放速度标签（selectbox 选中值）。用于速度映射查找 |
```

### 遗漏 2：`_play_speed` 默认值与 UI 默认不一致

**位置**：§2.4.1 vs §4.2.2

- §2.4.1：`_play_speed` 默认值 `0.5`（= 0.5 秒/步，对应 2x 速度）
- §4.2.2：selectbox 默认 index=2，对应标签 `1x`，值 `1.0`（秒/步）

两者矛盾：状态默认是 0.5（2x 速度），UI 默认是 1.0（1x 速度）。

**建议**：统一默认值。推荐统一为 `1x`（`_play_speed=1.0`），因为 1x 是直觉上的"正常速度"。

## 具体修改建议

### 修改 1：§1.2 对比表补充维度

**当前**（6 行）：
```
| 维度 | 浏览模式 | 回测模式 |
|------|---------|---------|
| 时间视角 | ... | ... |
| 数据窗口 | ... | ... |
| 高周期数据 | ... | ... |
| 预测显示 | ... | ... |
| 时间导航 | ... | ... |
| 数据源 | ... | ... |
```

**建议改为**（10 行）：
```
| 维度 | 浏览模式 | 回测模式 |
|------|---------|---------|
| 图表渲染 | 2×2 网格（日线/60分钟/15分钟/5分钟） | 2×2 网格（完全相同） |
| K 线 + 滤波 | 全量数据显示 | 窗口切片（≤n_pts 根 bar），渲染方式一致 |
| 时间视角 | 始终看到最新数据（含 day_offset） | 站在历史的某个时刻往前看 |
| 数据窗口 | day_offset 控制起点，始终显示最新 n_pts 根 bar | bar_index 控制窗口终点，严格屏蔽未来数据 |
| 高周期数据 | 从 Parquet 直读，最后一根可能已完成 | 可能需要合成未完成的高周期 bar |
| 预测显示 | 显示全部预测 | 只显示 bar_index 之前的预测对（消除前视偏差） |
| 时间导航 | day_offset + 前后按钮（侧边栏） | 隐藏 day_offset 控件；bar_index slider + 播放控件（主区域控制条） |
| 控制条 | 无 | ⏮ ◀ ▶/⏸ ▶▶ ⏭ + 速度选择 + 进度 slider |
| 滤波/触发计算 | _compute_filters + _compute_schmitt_trigger | 同一函数（数据子集不同但算法一致） |
| 数据源 | data/display/{tf}.parquet | 同一套 Parquet 文件 |
```

### 修改 2：§4.3.1 状态显示格式

**当前**：
```
📍 bar {bar_index + 1}/{total_bars} | {formatted_date}
播放中追加 | ▶ 播放中（绿色 markdown：:green[▶ 播放中]）
```

**建议改为**（与效果图对齐）：
```
📍 bar {bar_index + 1}/{total_bars}
📅 {formatted_date}
（播放中时追加第三行）
▶ 播放中  （绿色，:green[▶ 播放中]）
```

### 修改 3：§2.4.1 补充 `_play_speed_label` 键

在 `_play_speed` 行后添加：

```
| `_play_speed_label` | `str` | `"1x"` | 播放速度标签。对应速度选择器的选中值，用于 speed_map 查找 |
```

### 修改 4：统一 `_play_speed` 默认值

**§2.4.1 当前**：`_play_speed | float | 0.5`

**建议改为**：`_play_speed | float | 1.0`（与 §4.2.2 的 UI 默认值 `1x` 一致）

### 修改 5（可选）：§4.2.1 补充播放按钮状态切换说明

列 3（播放/暂停）的说明中追加：

> 按钮文案和图标根据 `_is_playing` 动态切换：`False` 时显示 ▶（播放），`True` 时显示 ⏸（暂停）。播放态按钮使用强调样式（如绿色背景），与普通按钮区分。

### 修改 6（可选）：§4.2.1 补充按钮分组说明

在表格上方或布局 ASCII 图中加入分组信息：
